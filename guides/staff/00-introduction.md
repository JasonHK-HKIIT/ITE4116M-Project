# 0. Introduction

## 0.1 About This Guide
Welcome to the VTC MyPortal Staff and Admin User Guide.

This guide explains end-to-end usage across both portal-facing features and dashboard management modules for authorized staff/admin users.

## 0.2 Intended Audience
This guide is for:
- Teaching and administrative staff
- System and module administrators
- Operational users responsible for managing student-facing content and records

## 0.3 Guide Objectives
This guide helps staff/admin users:
1. Navigate portal and dashboard interfaces effectively
2. Perform operational management tasks with consistency
3. Apply role/permission-aware workflows correctly
4. Resolve common issues and escalate with complete context

## 0.4 Scope of Staff/Admin Guide
This guide includes:
- Portal-side usage for staff/admin accounts
- Dashboard navigation and permission-based menus
- Calendar, academic structure, students, activities, news, resources, and system management modules
- Account quick menu and password change appendix topics

## 0.5 How to Use This Guide
Recommended learning path:
1. Read Chapter 01 to understand entry flow
2. Follow chapter order for complete onboarding
3. Use chapter numbers as quick references during daily operations
4. Review appendix chapters for account-level actions

Each chapter provides:
- Purpose and context
- Step-by-step task instructions
- Role/permission notes where applicable
- Troubleshooting guidance
- Escalation information

## 0.6 Role and Permission Notes
Some menu items and actions are role- or permission-dependent.

This guide indicates visibility differences such as:
- Staff vs admin-only sections
- Permission-gated module access
- Restricted operations in sensitive areas

Always verify account role before assuming access behavior.

## 0.7 Conventions Used in This Guide
For consistency and auditability:
- Numbered procedures define required sequence
- UI labels match on-screen text
- Workflow sections focus on operational tasks
- Troubleshooting is grouped by common failure scenarios

## 0.8 Security and Governance Reminder
Staff/admin users should apply strict operational controls:
- Use least-privilege access practices
- Protect account credentials and session security
- Validate critical data updates before save/publish
- Sign out from shared or unattended devices

## 0.9 Support and Escalation Baseline
When raising issues, prepare:
- Username and role
- Module/page and task attempted
- Input context and expected result
- Error message text
- Timestamp, screenshot, and browser/OS details

Providing complete context improves issue triage speed and resolution quality.

## 0.10 Version and Change Management
This guide should be reviewed after system updates, workflow changes, or role policy adjustments.

Keeping documentation synchronized with live behavior is essential for stable operations and training quality.
