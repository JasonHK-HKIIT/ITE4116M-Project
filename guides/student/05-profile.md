# 5. Profile

## 5.1 Purpose
This chapter explains how student users view profile information in VTC MyPortal.

The Profile section contains two pages:
1. Personal Particular
2. Programme and Modules

These pages are view-only and help students verify identity, contact details, programme assignment, and module information.

## 5.2 Access the Profile Section
You can access Profile from portal navigation links, such as the Profile entry under the Home page service area.

After opening Profile, use the available profile navigation to switch between:
- Personal Particular
- Programme and Modules

> Image placeholder: Profile entry point from portal home.

![Open profile from home](./images/05-profile-open-from-home.png)

## 5.3 Personal Particular Page
The Personal Particular page displays student identity and personal records in a structured two-column layout.

Page header:
- Profile
- Personal Particular
- Description text: View your personal information.

Right-side panel:
- Student photo/avatar

> Image placeholder: Personal Particular full-page overview.

![Personal particular overview](./images/05-profile-personal-overview.png)

## 5.4 Personal Particular Fields
The page may include the following fields:
- Student ID
- Chinese Name
- Family Name
- Given Name
- Institute
- Campus
- Current Class
- Gender
- Date of Birth
- Country
- Native Language
- Tel No.
- Mobile No.
- Correspondence Address in Hong Kong

Field behavior:
- If data is unavailable, the page shows a placeholder dash symbol.

> Image placeholder: Close-up of personal data rows.

![Personal particular fields](./images/05-profile-personal-fields.png)

## 5.5 How to Verify Personal Data
Recommended verification steps:
1. Confirm Student ID, Family Name, and Given Name first.
2. Check institute, campus, and current class for academic alignment.
3. Confirm contact details and correspondence address.
4. Review date of birth and language-related fields.

If any details appear incorrect, contact student administration for correction.

## 5.6 Programme and Modules Page
The Programme and Modules page shows your assigned programme records and linked modules.

For each programme card, the page shows:
- Programme name
- Programme Code
- A list of modules under that programme

For each module entry, the page shows:
- Module name
- Module Code

> Image placeholder: Programme and Modules page overview.

![Programme modules overview](./images/05-profile-programmes-overview.png)

## 5.7 Empty-State Messages
You may see the following messages depending on data assignment:
- No programmes assigned yet.
- No modules assigned for this programme.

These messages indicate there is currently no mapped record for your account in that section.

> Image placeholder: Empty-state examples for programme and module sections.

## 5.8 Typical Student Tasks
### Task A: Confirm Personal Identity Details
1. Open Personal Particular.
2. Check Student ID, Family Name, and Given Name.
3. Compare with official student records.

### Task B: Confirm Current Class and Institute
1. Open Personal Particular.
2. Check Current Class, Institute, and Campus.
3. Report mismatch to your programme office.

### Task C: Check Assigned Programme Modules
1. Open Programme and Modules.
2. Locate your programme card.
3. Review module names and module codes.

## 5.9 Troubleshooting
### Case A: Missing Personal Information
- Some fields may not yet be populated in central records.
- Confirm with administration if data should already exist.

### Case B: Photo Not Displaying
- Refresh the page.
- Retry with another supported browser.
- Report if image remains unavailable.

### Case C: No Programme or Module Listed
- Check for empty-state messages.
- Contact your academic office to confirm assignment status.

### Case D: Text Appears Different Language Than Expected
- The page uses localized programme/module names where available.
- If translation is unavailable, codes may be shown instead.

## 5.10 Data Accuracy and Security Notes
- Treat Profile information as sensitive personal data.
- Do not share screenshots containing personal information publicly.
- Verify profile details regularly, especially at semester start.

## 5.11 Support Information
When requesting support for profile issues, provide:
- Student ID
- Affected page name (Personal Particular or Programme and Modules)
- Specific field or programme/module that appears incorrect
- Screenshot of the issue
- Browser and device details
